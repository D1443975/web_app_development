# 產品需求文件（PRD）— 個人記帳簿系統

---

## 1. 專案概述

### 1.1 背景與動機

在日常生活中，每個人都需要管理自己的收入與支出，但許多人缺乏有效的工具來追蹤每一筆交易。手動記帳容易遺漏且不方便統計，市面上的記帳 App 又經常功能過於複雜或需要付費。本專案旨在打造一個**簡潔、易用的個人記帳簿網頁系統**，讓使用者可以輕鬆記錄每日的收支，並透過直觀的統計功能掌握自己的財務狀況。

### 1.2 目標用戶

- **所有人**：任何需要管理個人財務的使用者，不限年齡與職業。
- 特別適合：學生、上班族、自由工作者等，希望以簡單方式追蹤日常花費的人。

### 1.3 核心價值主張

| 價值 | 說明 |
|------|------|
| 📝 簡單記帳 | 快速記錄每筆收入與支出，操作直覺不繁瑣 |
| 📊 即時統計 | 即時計算餘額，掌握財務現況 |
| 📂 分類管理 | 依照類別分組，了解錢花在哪裡 |
| 📅 月度報表 | 按月統計花費，追蹤消費趨勢 |
| 🌐 網頁操作 | 透過瀏覽器即可使用，不需安裝任何軟體 |

## 2. 功能需求

### 2.1 主要功能列表

| 編號 | 功能名稱 | 簡述 |
|------|----------|------|
| F1 | 紀錄收入 | 使用者可新增一筆收入紀錄，包含金額、類別、日期與備註 |
| F2 | 紀錄支出 | 使用者可新增一筆支出紀錄，包含金額、類別、日期與備註 |
| F3 | 計算餘額並顯示 | 系統自動計算總收入減總支出，於首頁顯示即時餘額 |
| F4 | 計算每個種類花費 | 依照類別彙整支出金額，以列表或圖表呈現各類別的花費比例 |
| F5 | 統計每月花費 | 按月份彙整收支資料，顯示每月的收入、支出與結餘變化 |
| F6 | 收支紀錄列表 | 顯示所有收支紀錄，支援依日期排序與篩選 |
| F7 | 編輯 / 刪除紀錄 | 使用者可修改或刪除已存在的收支紀錄 |

### 2.2 使用者故事（User Stories）

#### F1 — 紀錄收入
> 作為 **一般使用者**，我希望 **能夠新增一筆收入紀錄（包含金額、類別、日期、備註）**，以便 **完整追蹤我的所有收入來源**。

#### F2 — 紀錄支出
> 作為 **一般使用者**，我希望 **能夠新增一筆支出紀錄（包含金額、類別、日期、備註）**，以便 **清楚知道每一筆錢花在哪裡**。

#### F3 — 計算餘額並顯示
> 作為 **一般使用者**，我希望 **在首頁能看到目前的總餘額（總收入 − 總支出）**，以便 **隨時掌握自己的財務狀況**。

#### F4 — 計算每個種類花費
> 作為 **一般使用者**，我希望 **能查看各類別（如飲食、交通、娛樂等）的花費統計**，以便 **了解自己的消費分佈，進而調整開支習慣**。

#### F5 — 統計每月花費
> 作為 **一般使用者**，我希望 **能按月份查看收入與支出的統計資料**，以便 **追蹤每月消費趨勢，做好預算規劃**。

#### F6 — 收支紀錄列表
> 作為 **一般使用者**，我希望 **能瀏覽所有收支紀錄的清單，並依日期排序或篩選**，以便 **快速找到特定的交易紀錄**。

#### F7 — 編輯 / 刪除紀錄
> 作為 **一般使用者**，我希望 **能修改或刪除已經輸入的收支紀錄**，以便 **更正錯誤或移除不需要的資料**。

## 3. 非功能需求

### 3.1 技術限制

| 項目 | 技術選擇 |
|------|----------|
| 後端框架 | Flask (Python) |
| 模板引擎 | Jinja2 |
| 資料庫 | SQLite |
| 前端 | HTML + CSS + JavaScript（Jinja2 模板渲染） |

### 3.2 效能考量

- 頁面載入時間應在 **2 秒以內**
- 資料庫查詢應針對常用欄位（日期、類別、類型）建立索引
- 統計計算應在伺服器端完成，避免前端處理大量資料

### 3.3 安全考量

- 所有使用者輸入需進行 **輸入驗證**（金額為正數、日期格式正確等）
- 使用 Flask 內建的 **CSRF 保護**
- 防止 **SQL Injection**：使用 ORM 或參數化查詢
- 防止 **XSS 攻擊**：Jinja2 預設自動轉義 HTML

### 3.4 可用性考量

- 介面應支援 **響應式設計（RWD）**，在手機與桌面瀏覽器皆可正常使用
- 操作流程簡潔，新增一筆紀錄不超過 **3 個步驟**
- 重要數據（餘額、本月花費）應在首頁一目了然
